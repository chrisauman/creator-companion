Mobile design notes:
  • Top bar (dark) — hamburger (left) opens the drawer; brand (center); compose pill (right, cyan) opens inline composing.
  • Greeting + horizontal stats — same content as desktop in a horizontally scrolling card strip. Hybrid progress reward (tier badge + bar) lives on the streak card.
  • Today panel stacks vertically: Spark hero → small prompt → mood grid (4×3, 12 moods, no "+ more" needed if the screen is tall enough — shown 2×4 here only to fit the preview) → "Just begin" card. Below is the entry list.
  • Reading view (screen 2) — full-screen overlay sliding over from the right. Back chevron returns to the list (preserves scroll). Edit button takes you to inline edit (same overlay pattern). Side gesture: swipe right to go back.
  • Composing (screen 3) — full-screen overlay. Cancel button (left), date pill (center, paid users), Save button (right). Prompt banner at top. Toolbar at bottom for mood / photo / tag / format.
  • Drawer (screen 4) — left slide-in carrying the same nav as desktop sidebar (Journal, Notifications, To Do, Sparks, Trash) plus the user card at the bottom with name + tier + gear + logout.
  • Bottom nav stays — primary one-tap switching between Journal / Alerts / To Do / Sparks / Account. The drawer is for secondary nav (Trash, settings, sign out).
Open questions:
  • Mood grid: 4×3 (all 12 moods, 4 wide) — fits cleanly. Or 6×2 like desktop? 4×3 reads better on narrow phones.
  • Stats: horizontal scroll vs. 2×2 grid? Horizontal lets you see the streak with progress reward without sacrificing space; 2×2 is more compact but smaller numbers.
  • Is keeping both bottom nav AND drawer the right call, or should we pick one?
  • Reading/compose overlays: native iOS slide-over (from right, with chevron) vs modal that fades in?
